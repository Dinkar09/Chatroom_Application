Self-Electing Hub — Port Binding Decision Flow
🔌 start_master_hub_background() — First Peer Wins
Step 01 · Any Peer
Attempt Port Bind
Every peer that selects "Create Room" runs asyncio.start_server() targeting 127.0.0.1:8888. The OS grants the bind to exactly one peer — whoever calls it first.
asyncio.start_server(handler, '127.0.0.1', 8888)
→ if OSError
Step 02 · Already Bound
OSError Caught (10048)
If port 8888 is already held by another peer-hub, the OS raises an OSError. This is not a crash — it is the deliberate detection signal: a Hub already exists.
except OSError: → route to existing hub
→ both paths
Step 03 · Either Outcome
Connect as Client
After the bind attempt (success or failure), the peer waits 100ms then connects to 127.0.0.1:8888 as a standard client to send the CREATE command.
await asyncio.sleep(0.1) → open_connection()
→ result
Step 04 · Running State
Hub Runs as Background Task
The winner's Hub runs via asyncio.create_task(hub.serve_forever()) — non-blocking, so the same process also runs the client chat UI concurrently.
asyncio.create_task(hub.serve_forever())

OSI Model — Protocol Layer Breakdown
L7
App
Application
Custom text protocol: CREATE, JOIN, ACCEPTED, DENIED, /accept, /deny. Newline-delimited frames. History sync block markers.
L6
Pres
Presentation
UTF-8 encoding via .encode() / .decode(). Newline \n as frame delimiter. readline() at receiver strips framing.
L4
Tran
Transport
TCP — asyncio StreamReader/StreamWriter wrap a reliable, ordered, byte-stream socket. Port 8888. drain() ensures kernel buffer flush before next write.
L3
Net
Network
IPv4 loopback address 127.0.0.1. All traffic stays within the local machine's loopback interface — no physical NIC traversal.
