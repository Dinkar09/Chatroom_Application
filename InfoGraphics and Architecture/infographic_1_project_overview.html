Wire Protocol — Message Contract
📤 Client → Hub
CREATE <room> <user>
Initialises a new virtual room, appoints sender as Admin
JOIN <room> <user>
Requests entry; triggers Future-based admin authorization gate
/accept <username>
Admin sets Future → True, unblocks pending JOIN coroutine
/deny <username>
Admin sets Future → False, client receives DENIED and disconnects
📥 Hub → Client
ACCEPTED_ADMIN
Room created successfully — sender enters Admin chat loop
ACCEPTED
JOIN authorised — history sync begins, then live chat loop starts
DENIED: <reason>
Access refused — room full, not found, or rejected by Admin
ACCESS REQUEST
Hub notifies Admin of a pending JOIN with accept/deny instructions
⚡ Broadcast Messages
🌟 Created room
Emitted to room on first Admin connect
👋 User joined
Broadcast when ACCEPTED peer enters the room
🚪 User left
Emitted on clean disconnect or EOF
⚠️ Admin gone
Emitted when Admin disconnects — room frozen
[User (Admin)]: msg
Standard chat — role tag appended for Admin messages
Core Features
⚡
Async I/O
Non-blocking StreamReader / StreamWriter via asyncio.gather for concurrent send & receive
🏠
Virtual Rooms
Multiple isolated chatrooms on one hub; each with its own member map and history log
🔐
Auth Gate
asyncio.Future suspends JOIN until Admin resolves via /accept or /deny command
📜
History Sync
Newly admitted users receive the full past message log before entering live chat
🔁
Self-Hosting
First peer silently binds port 8888 as Hub; subsequent peers detect OSError and route to it
🧹
Clean Lifecycle
finally block handles disconnect, room cleanup, and Admin-gone broadcast automatically
Core Data Structures
dict
rooms
dict[str, dict[StreamWriter, str]]
# { "room_name": { writer: "username" } }
Primary room registry. Maps each room name to its member table, where keys are StreamWriter objects and values are usernames. Enables O(1) broadcast iteration and presence checks.
dict
room_histories
dict[str, list[str]]
# { "room_name": ["msg1", "msg2", ...] }
Append-only message log per room. Every broadcast appends here first. Replayed sequentially to newly authorised peers during the history sync phase before live mode.
dict
room_admins
dict[str, StreamWriter]
# { "room_name": admin_writer }
Maps each room to its Admin's StreamWriter. Used to forward JOIN notifications and to identify Admin in the command loop. Cleared on Admin disconnect, freezing the room.
dict
pending_authorizations
dict[str, dict[str, Future]]
# { "room": { "username": asyncio.Future } }
The authorization gate. Each pending JOIN creates a Future that suspends the connection coroutine. /accept sets result(True), /deny sets result(False). Resolved futures are deleted immediately.
